关于git的相关应用：

总结一下时光机：
1、HEAD指向的版本就是当前版本，因此，git允许我们在版本的历史之间穿梭，使用命令：、
git reset --hard commit_id
2、穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。
3、要重返未来，用git reflog 查看历史命令，以便确定要回到未来的哪个版本。

暂存区：
是很重要的一部分。

git是如何进行跟踪修改的？ 
git针对的是修改，而不是文件。只有将文件 git add 文件名， 才能进行提交。

撤销修改：
场景一：当你乱改了工作区某个文件的内容时，想直接丢弃工作区的修改时，用命令git restore 文件名
场景二：当你不但乱改了工作区某个文件的内容时， 还添加到了暂存区，想丢弃修改，分两步，第一步用命令 git restore --staged 文件名
就回到了场景一，按照场景一操作。
场景三：已经提交到不合适更改的版本库时，可以使用版本回退 一节，不过前提是没有推送到远程库。

删除：
命令git rm用于删除一个文件，如果一个文件已经被提交到版本库，那么永远不用担心误删，但是只能恢复到最新版本，你会丢失最后一次提交后你修改的内容。

远程仓库使用方法：
将自己电脑的key部署到github上。

添加远程仓库：
要关联一个远程库，使用命令： git remote add wuhqing git@github.com:wuhq1/kgdemo.git
关联后，使用命令：git push -u wuhqing master第一次推送master分支的所有内容；
伺候，每次本地提交之后，只要有必要，就可以使用命令 git push origin master推送最新修改。

对于分支：
查看分支：git branch
创建分支：git branch <名字>
创建并切换分支：git switch -c <name>
合并某分支到当前分支：git merge <name>
删除分支：git branch -d <name>
